# Task: Web Push goal notifications

Implement "notify me when a goal is scored in a live match" using Web Push.
Read `../CLAUDE.md` first for project context. Conventions: Go stdlib + layered
`internal/*`, providers behind interfaces, graceful degradation, **never commit
`.env`**, commits authored **Scroti** (`git config user.name Scroti`). Build +
`go vet` + `go test ./...` before committing. Service: systemd `oddvice-api` in
`/opt/oddvice-api`, binds `127.0.0.1:8080` behind Caddy; restart with
`sudo systemctl restart oddvice-api`.

## Already in place (reuse, don't rebuild)
- `internal/teams` (api-football Pro, season 2026): `Service.Live(ctx)` returns
  `[]LiveMatch{FixtureID, Home, Away, HomeGoals, AwayGoals, Elapsed, Status}`.
- The web service worker (`public/sw.js` in the **oddvice** web repo) ALREADY
  handles the `push` and `notificationclick` events. Push payload it expects:
  `{ "title": "...", "body": "...", "url": "/..." }`.

## VAPID (one-time, already generated by the user)
```
VAPID_PUBLIC=<public key>
VAPID_PRIVATE=<private key>
VAPID_SUBJECT=mailto:you@oddvice.app
```
Public key → web env (`NEXT_PUBLIC_VAPID_KEY`). Private key + subject → API `.env`
(gitignored). Add them to `internal/config` (a `Push` struct) and `.env.example`
(empty placeholders only).

## API side (this repo: oddvice-api)
1. `go get github.com/SherClockHolmes/webpush-go`.
2. New `internal/push` package:
   - **Store** — persist subscriptions to a JSON file (default
     `/opt/oddvice-api/data/push-subs.json`, path via env `PUSH_STORE_PATH`).
     Load on startup; `Add(sub)` dedupes by `endpoint`; `All()`; `Remove(endpoint)`.
     Mutex-guarded; create the dir if missing; tolerate a missing/empty file.
   - **Sender** — `Send(sub, payloadJSON)` via `webpush.SendNotification` with the
     VAPID keys + subject. On a `404`/`410` response, drop that subscription
     (it's gone). Best-effort; log failures.
3. **Handler** (new `internal/push` handler, registered in `server.go`):
   - `POST /api/v1/push/subscribe` — body is the browser `PushSubscription` JSON
     (`{endpoint, keys:{p256dh, auth}}`). Store it. Return 204.
   - `POST /api/v1/push/unsubscribe` — body `{endpoint}`. Remove. Return 204.
   - (optional) `GET /api/v1/push/public-key` — returns the VAPID public key, so
     the web doesn't have to hardcode it. Nice-to-have.
4. **Goal watcher** — a goroutine started from `server.New` (pass it the teams
   `Service` + push `Store`/`Sender`; start only if VAPID is configured):
   - Every ~30s call `teams.Service.Live(ctx)`.
   - Keep `map[fixtureID]{home,away int}` of last-seen scores.
   - When a fixture's score **increases**, that's a goal → build a payload
     `{title: "GOAL! <Home> <h>-<a> <Away>", body: "<scoring side> scored · <elapsed>'", url: "/"}`
     and `Send` to every stored subscription. (Score-based detection — no need for
     events/Claude. You MAY later enrich the scorer name via
     `apifootball` events, but keep v1 score-based.)
   - Forget a fixture once it's no longer live (so a later match reusing memory
     doesn't misfire). Guard against the first observation of a match (don't push
     for the initial score, only for increases after we've seen it).
   - Make it cancellable via the server's context; respect the existing
     rate-limit/cooldown behaviour in the apifootball client.
5. Wire the watcher + handler in `server.registerFeatures` / `New`. If VAPID isn't
   configured, skip the watcher and have `/subscribe` return 503 (so the feature
   is a no-op until keys are set).

## Web side (oddvice web repo)
1. `NEXT_PUBLIC_VAPID_KEY` in `.env.local` (+ `.env.example`).
2. A **"Notify me of goals"** toggle (good spot: under the live scoreboard, or in
   `/profile`). On enable:
   - `await Notification.requestPermission()` → if granted,
   - `const reg = await navigator.serviceWorker.ready;`
   - `const sub = await reg.pushManager.subscribe({ userVisibleOnly: true, applicationServerKey: urlBase64ToUint8Array(NEXT_PUBLIC_VAPID_KEY) });`
   - `POST ${NEXT_PUBLIC_API_URL}/api/v1/push/subscribe` with `sub.toJSON()`.
   - Persist the on/off state in `localStorage`; on disable, `sub.unsubscribe()` +
     POST `/unsubscribe`.
   - Include the standard `urlBase64ToUint8Array` helper.
3. The SW already displays the notification — verify the payload shape matches
   (`title`, `body`, `url`). No SW change should be needed.

## Acceptance
- Subscribe from the web (permission granted) → subscription stored on the VPS.
- When a live match's score increases, subscribers get a "GOAL!" notification
  within ~30s, tapping it opens the app.
- Everything degrades gracefully when VAPID isn't set (no crashes, feature off).
- `go build ./... && go vet ./... && go test ./...` clean; commit as Scroti.
